# 출처 갱신 프로토콜

## 갱신 등급

| 등급 | 대상 | 갱신 주기 | 예시 |
|---|---|---:|---|
| A | 가격, 금리, 환율, 공시, 실거래가 | 당일 | 주가, 채권 금리, 기준금리, 실거래가 |
| B | 시장 통계, 섹터 지표, 거시 지표 | 7일 | 물가, 고용, 거래량, 재고 |
| C | 법률, 규정, 세금, 감독 지침 | 30일 또는 사건 발생 시 | 금소법, 세법, 부동산 규제 |
| D | 투자 교육 자료, 일반 개념 | 180일 | 분산투자, 리밸런싱, 위험 설명 |

## 보고서 작성 전 절차

1. `funmemory/sources/source_registry.yaml`에서 필요한 출처를 고른다.
2. 각 출처의 최신 게시일, 시행일, 데이터 기준일을 확인한다.
3. 보고서에 `source_checked_at`, `data_as_of`, `staleness_risk`를 기록한다.
4. 오래된 데이터는 결론에 쓰지 않고 `추가 확인 필요`로 분리한다.
5. 법률과 세무는 단정형 결론을 피하고 이슈, 근거, 확인 질문으로 정리한다.

## 출처 우선순위

1. 원문 법령, 감독기관, 중앙은행, 거래소, 공시 시스템.
2. 공식 API 또는 공식 통계 포털.
3. 감사된 재무제표, 투자설명서, 사업보고서.
4. 신뢰 가능한 리서치. 단, 투자 판단 근거로 쓸 때는 원자료를 재확인한다.
5. 뉴스와 블로그는 힌트로만 사용하고 결론 근거로 단독 사용하지 않는다.
