# Verify 평점 계약

## 목적

전략 수립, 자료 분석, 최종 리포트 작성 시 체크리스트를 단순 확인으로 끝내지 않고 점수 기반으로 반복 검증한다.

## 점수 체계

각 세부 항목은 0점에서 10점으로 평가한다. 카테고리와 최종 결과는 100점 만점으로 환산한다.

| 점수 | 의미 |
|---:|---|
| 0 | 누락 |
| 2 | 매우 부족 |
| 4 | 보완 필요 |
| 6 | 최소 사용 가능 |
| 8 | 양호 |
| 10 | 강함 |

카테고리 점수는 `항목 평균 x 10`으로 100점 환산한다.

- 각 점수는 확인한 `evidence_id` 또는 원천 데이터 필드와 연결한다. 근거 없는 자기평가 점수는 0점으로 처리한다.
- `전략 절차 점수`, `실현 예측 품질`, `실행 품질`, `리스크 준수`를 분리한다. 절차 점수로 적중률이나 수익성을 대체하지 않는다.
- 100점은 최신 필수 데이터 완비, unknown 0개, matured_unscored 0개, 중대 차단 조건 0개, 모든 항목의 증거 연결을 모두 만족할 때만 허용한다.
- 실현 예측 표본이 부족하면 예측 품질은 `N/A`이며, 절차 점수가 100이어도 만점 전략·최고 적중·수익 보장으로 표현하지 않는다.

## 반복 검증 루프

1. 초안 작성 후 1차 Verify를 수행한다.
2. 기준 미달 카테고리를 수정한다.
3. 수정본에 2차 Verify를 수행한다.
4. 여전히 기준 미달이면 최대 3차까지 수정한다.
5. 3차 후에도 미달이면 결론을 `보류` 또는 `중단 권고`로 낮추고 부족한 데이터와 다음 확인 액션만 제시한다.
