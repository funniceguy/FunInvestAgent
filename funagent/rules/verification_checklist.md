# 검증 체크리스트

## 평점 계약

- [ ] 각 항목을 0-10점으로 평가했는가.
- [ ] 카테고리 점수를 100점 환산으로 기록했는가.
- [ ] 전체 가중 점수와 통과 기준을 비교했는가.
- [ ] 최소 2회 이상 반복 검증했는가.
- [ ] 기준 미달 항목의 수정 내역을 기록했는가.
- [ ] 중대 차단 조건 발생 여부를 기록했는가.
- [ ] 각 점수를 evidence_id 또는 원천 데이터 필드와 연결했는가.
- [ ] 절차 점수와 실현 예측·실행·리스크 점수를 분리했는가.
- [ ] unknown 또는 matured_unscored가 있는데 100점을 부여하지 않았는가.
- [ ] 최종 판정이 점수와 차단 조건에 맞게 낮춰졌는가.
- [ ] 금일/내일/차주 전략이면 금주 보고서와 리뷰를 검색해 예측 복기·평점 보정 적합성을 평가했는가.
- [ ] 금주 예측 복기·평점 보정 적합성 점수가 기준 미달이면 진행 가능으로 표시하지 않았는가.
- [ ] 이전 전략 실패·지연·무효화가 언급되면 전망 실패 사후분석 적합성 점수가 기준 미달일 때 진행 가능으로 표시하지 않았는가.
- [ ] 차주/미래 전망 전략의 검색·발췌·분석 적합성 점수가 기준 미달이면 진행 가능으로 표시하지 않았는가.
- [ ] 내일/금일/차주/한 주 방향 예측의 예측 보정·적중 추적 적합성 점수가 기준 미달이면 진행 가능으로 표시하지 않았는가.
- [ ] 금리, 환율, 정부정책, 세계정세, 최대 효율 후보가 포함되면 거시 원인 민감도 매트릭스 적합성 점수가 기준 미달일 때 진행 가능으로 표시하지 않았는가.
- [ ] 투자자 심리 게이트를 통과하지 못한 전략을 진행 가능으로 표시하지 않았는가.
- [ ] 공격적 단기 매매 적합성 점수가 기준 미달이면 신규 진입을 진행 가능으로 표시하지 않았는가.
- [ ] 장기 전략이면 장기 전용 가중 점수 95점 이상, 목표수익률 현실성 100점, 표현 안전성 100점을 충족했는가.
- [ ] 모든 실행 전략에서 자본보전·손실회복 적합성 97점 이상과 게이트 clear를 충족했는가.

## 입력 검증

- [ ] 사용자 목표가 명확한가.
- [ ] 투자 기간이 명확한가.
- [ ] 위험 감내도가 기록되어 있는가.
- [ ] 비상금과 1년 내 확정 지출을 분리했는가.
- [ ] 부채와 금리를 확인했는가.
- [ ] 계좌·슬리브·1회 거래 최대 손실과 감당 가능한 회복기간을 확인했는가.

## 자본보전·손실회복 검증

- [ ] 생활·비상·확정지출 자금을 위험자산에서 분리했는가.
- [ ] 계좌·슬리브·거래·하루·주간 손실예산을 해당 기간에 맞게 기록했는가.
- [ ] 손절가가 보장 체결가가 아님을 반영해 갭·슬리피지·거래정지 스트레스를 계산했는가.
- [ ] 계획 손절과 스트레스 손실 중 큰 값으로 포지션 상한을 계산했는가.
- [ ] 종목·섹터·지수·환율·금리 기준의 상관·중복 노출을 합산했는가.
- [ ] 최대 낙폭 후 필요한 회복수익률을 `d / (1 - d)`로 계산했는가.
- [ ] 회복기간을 보수·기본·강세 범위로 표시하고 보장하지 않았는가.
- [ ] 회복 근거를 이익·FCF, 재무, 논리, 밸류에이션, 유동성·공급 중 3개 이상으로 검증했는가.
- [ ] 구조적 훼손을 일시적 하락과 분리하고 물타기·회복 대기를 차단했는가.
- [ ] 비용·세금·슬리피지 후 기대값이 양수인가.
- [ ] 자본보전 게이트가 watch_only 또는 blocked이면 랭킹·신규 진입·비중 확대를 금지했는가.
- [ ] 절차적 안정성을 원금·수익·빠른 회복 보장으로 표현하지 않았는가.

## 데이터 검증

- [ ] 가격, 금리, 법령, 공시는 최신 출처로 확인했는가.
- [ ] 데이터 기준일을 적었는가.
- [ ] 출처 ID를 기록했는가.
- [ ] 추정과 사실을 분리했는가.
- [ ] 차주/미래 전망에 사용한 출처의 source_id, published_at, data_as_of, source_checked_at을 기록했는가.
- [ ] 내일/주간 예측에 필요한 지수, 수급, 프로그램, 선물, USD/KRW, 해외 선행시장, 금리·달러, 이벤트 캘린더 수치를 기록했는가.
- [ ] 폭락, 급락, 반등 단타, 반도체 대형주, ETF/ETN, 레버리지, 기관·프로그램 매도 전략에 필요한 기관 선물, 프로그램 차익/비차익, 사이드카, ETF/ETN 괴리율, 단일종목 레버리지 리밸런싱 데이터를 기록했는가.
- [ ] 세력·정치 압박·조작·전쟁 단독 원인·시장 전체 자금 이탈·ADR/원주·±2배가 언급되면 기관 세부, 시장 폭·집중도, 해외 동종시장, 공식 조사 상태, 설정·환매·헤지 자료를 기록했는가.
- [ ] 요약, 근거 카드, 후보 설명의 종목·ETF·ETN·ADR을 한글명/공식 풀네임 (티커) 형식으로 표시했는가.
- [ ] 실제 상품명, 티커, 거래소, 통화, 상품 유형을 기록했는가.
- [ ] 최우선 매수호가, 최우선 매도호가, 호가 잔량, 스프레드 %, 기준 시각을 기록했는가.
- [ ] 거래량, 거래대금, 20일 평균 거래대금 또는 유동성 대체 지표를 기록했는가.
- [ ] 저평가 우량주/GARP/성장 품질 후보의 OPM 또는 금융 업종별 대체 수익성 지표를 기록했는가.
- [ ] ETF/ETN이면 NAV/IOPV, 괴리율, 총보수, 시장조성 상태를 기록했는가.
- [ ] 호가 미확인 또는 지연 데이터일 때 신규 주문 결론을 낮췄는가.

## 전략 검증

- [ ] 보수, 기준, 공격 시나리오를 구분했는가.
- [ ] 이전 전략의 가정, 실제 결과, 오차, 최초 경고 신호, 재발 방지 규칙을 기록했는가.
- [ ] 이전 전략의 무효화 조건이 깨졌다면 같은 논리의 신규 진입, 물타기, 비중 확대를 차단했는가.
- [ ] 차주/미래 전망은 기본/강세/약세 시나리오와 무효화 조건을 포함했는가.
- [ ] 금주 이전 prediction_id와 예측 시나리오를 성공/부분 성공/실패/판정 불가로 분류했는가.
- [ ] 반복 실패 원인을 다음 전략의 최소 점수, 필수 데이터, 차단 조건 강화안으로 변환했는가.
- [ ] 국내·미국 전체 보고서와 리뷰를 검색하고 prediction_id 원장을 대조했는가.
- [ ] legacy_unscorable과 matured_unscored를 숨기지 않았는가.
- [ ] 시장·기간·국면별 성과와 예측·실행·리스크 결과를 분리했는가.
- [ ] 표본 부족 또는 단기 성공 1회만으로 Verify 기준을 낮추지 않았는가.
- [ ] 내일/주간 예측은 상승/혼조/하락 확률, 예상 범위, 핵심 수치, 충돌 근거, 검증 시각을 포함했는가.
- [ ] prediction_id, 실제 판정 기준, 성공/실패 기준, 오차 계산 방식을 기록했는가.
- [ ] 외국인·기관·프로그램·환율 중 2개 이상이 충돌하면 주방향 확률을 60% 이하로 제한했는가.
- [ ] 기계적 수급 게이트가 watch_only 또는 blocked이면 주방향 확률과 신규 진입 결론을 낮췄는가.
- [ ] 폭락 원인을 구조적 취약성·방아쇠·증폭기·결과·반대 가설로 분리하고 동일 주문·사실을 중복 합산하지 않았는가.
- [ ] 기관 합계를 금융투자·투신·사모·연기금 등으로 분해하고 정치·연기금 가설의 실제 기여액을 비교했는가.
- [ ] 비차익 프로그램 총계를 특정 레버리지 상품이나 불법 알고리즘의 정확한 주문액으로 귀속하지 않았는가.
- [ ] +2배/-2배 양방향 일일 재설정과 동방향 헤지 가능성, 설정·환매·스왑·선물 귀속 한계를 기록했는가.
- [ ] 전쟁·유가·정책을 해외 동종주·다른 지수·시장 폭과 비교하고 단독 원인인지 trigger/amplifier인지 구분했는가.
- [ ] ADR/원주 괴리의 전환·결제·세금·환율·차입 제약과 broad_exit/sector_rotation/mixed_exit_rotation을 판정했는가.
- [ ] 정치·시세조종을 공식 확인·조사·공개 수급·미입증으로 등급화하고 공식 발표 부재를 조작 부재 증명으로 쓰지 않았는가.
- [ ] 호가 잔량을 복수 스냅샷, 실제 체결, VWAP/AVWAP, 경매·공시 공급으로 교차검증했는가.
- [ ] 미국 depth의 venue coverage와 한국 KRX/NXT 세션을 구분했는가.
- [ ] 증자·ATM·CB/BW·워런트·보호예수·내부자·리밸런싱 잠재 공급을 확인했는가.
- [ ] 급등·순환·구조적 주도·반등·방어·분산 클래스 확률과 신뢰도를 기록했는가.
- [ ] 현재 사이클 단계와 다음 전환·무효화 조건을 전략에 반영했는가.
- [ ] 외국인 현물 순매수 신호를 기관 선물, 프로그램, ETF/ETN, 단일종목 레버리지와 교차검증했는가.
- [ ] 세계정세와 경제지표를 금리, 환율, 원자재, 수급, 이익, 밸류에이션, 위험선호 전달 경로로 연결했는가.
- [ ] 금리, 환율, 정부정책, 세계정세, 원자재, 유동성, 섹터 선행 지표를 후보별 민감도로 점수화했는가.
- [ ] 최대 효율 후보를 거시 순풍, 실적 촉매, 가격 반응, 수급, 유동성, 손익비, 상관 리스크, 이벤트 타이밍으로 점수화했는가.
- [ ] 강한 역풍 요인이 2개 이상인 후보를 신규 진입 또는 비중 확대에서 제외했는가.
- [ ] 실행 조건과 중단 조건이 있는가.
- [ ] 상품별 주문 방식, 지정가 기준, 체결 가능성 판정이 있는가.
- [ ] 세금, 수수료, 환율, 유동성 비용을 언급했는가.
- [ ] 단일 자산 집중 리스크를 확인했는가.
- [ ] 공격적 단기 매매의 기대치, 가격 반응, 거래대금, 기관/외국인/프로그램 수급, 환율, 상대강도, 손익비를 점수화했는가.
- [ ] 공격적 단기 매매 전에 기계적 수급 게이트의 `mechanical_flow_clear/caution/watch_only/blocked` 판정을 확인했는가.
- [ ] 수급과 환율이 단기 방향성과 충돌하면 watch_only 또는 blocked로 낮췄는가.
- [ ] 저마진, 낮은 ROE, 약한 영업현금흐름, 부채·이자 부담, 성장 촉매 부재, 주주환원 부재, 약한 수급 7개 위험 신호를 카운트했는가.
- [ ] 위험 신호 4개 이상 후보를 랭킹에서 제외하고, 2-3개 후보는 보류/관찰로 낮췄는가.

## 장기투자·배당·커버드콜 검증

- [ ] 월 목표수익률을 `(1 + m)^12 - 1`로 유효 연수익률에 복리 환산했는가.
- [ ] 월 10% 반복 목표를 유효 연 213.84%로 표시하고 장기 코어 기본 시나리오에서 차단했는가.
- [ ] 목표를 현금 분배, 가격 상승, 세전 총수익, 세후 총수익으로 구분했는가.
- [ ] 투자기간, 월 납입·인출, 비상금, 최대 낙폭, 세금 계좌를 확인했는가.
- [ ] 최근 5년·직전 12개월·향후 2-3년의 이익과 FCF를 구분했는가.
- [ ] 정상화 EPS·FCF 없이 trailing PER/PBR을 장기 코어 근거로 쓰지 않았는가.
- [ ] ROE/ROIC, 자본비용, 순부채, 이자보상, 3년 누적 희석과 순자사주 매입을 확인했는가.
- [ ] 기대 총수익을 이익 성장, 배당, 순자사주 매입, 멀티플 변화, 환율, 비용·세금으로 분해했는가.
- [ ] 분배율, trailing distribution rate, 30-day SEC yield, 배당수익률, 옵션 프리미엄, 총수익률을 분리했는가.
- [ ] 커버드콜의 기초자산, 옵션 기초, 매도 비율, ATM/OTM, 만기, 롤 규칙을 확인했는가.
- [ ] 분배 재원을 순투자소득, 순옵션 손익, 실현 자본이득, 원금환급으로 분해했는가.
- [ ] 분배 전후 주당 NAV 추세와 분배 재투자 총수익을 동시에 확인했는가.
- [ ] 동일 기초자산 일반 ETF와 같은 기간·통화·세전/세후 기준 총수익을 비교했는가.
