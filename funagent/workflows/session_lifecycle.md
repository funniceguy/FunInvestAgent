# 세션 라이프사이클

## 1. Intake

- 사용자 요청을 한 문장으로 재정의한다.
- 산출물 종류를 정한다.
- 부족한 필수 입력을 확인한다.

## 2. Memory Load

- 관련 도메인 메모리 파일을 읽는다.
- 필요한 출처 ID를 고른다.
- 법률/규제 가드레일을 적용한다.

## 3. Analysis

- 개인 재무 체력.
- 시장 관찰.
- 포트폴리오 적합성.
- 법률 및 세금 이슈.

## 4. Strategy

- 보수, 기준, 공격 시나리오.
- 실행 조건과 보류 조건.
- 리밸런싱 규칙.

## 5. Verification

- 출처 최신성.
- 필수 섹션.
- 과잉 확신 표현.
- 법률/세무 단정 표현.

## 6. Output

- `funoutput/templates` 중 하나를 사용한다.
- 산출물을 `funoutput/reports` 또는 `funoutput/sessions`에 저장한다.
- 세션 로그를 갱신한다.
