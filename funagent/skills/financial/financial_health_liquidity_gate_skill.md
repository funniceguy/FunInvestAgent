# 스킬: 재무 체력·유동성 게이트

## 목적

투자 상품을 보기 전에 현금흐름, 부채, 비상자금, 확정지출과 보험·세금 제약을 수치화해 투자 가능 자금과 최대 손실예산을 정한다.

## 필수 입력

- 월 순소득, 필수·선택 지출, 부채 원리금, 소득 변동성과 부양가족.
- 유동 현금, 보호 예금, 1년 내 확정지출, 비상자금 목표 개월 수.
- 부채 잔액, 실효금리, 변동·고정, 만기, 중도상환비용과 담보.
- 보험·연금·세금·법률상 확정 의무.
- 목표별 날짜·금액·기준 통화와 최대 허용손실·회복기간.

## 계산

- 가용 유동자금 = 유동자산 - 1년 내 확정지출.
- 비상자금 충족 개월 = 가용 유동자금 / 월 필수지출.
- 부채상환부담 = 월 부채 원리금 / 월 순소득.
- 순저축률 = 월 순저축 / 월 순소득.
- 투자 가능 자금 = 총 가용자금 - 비상자금 부족액 - 확정지출 - 사용자 지정 안전 버퍼.

각 기준값은 사용자의 고용·가족·보험·소득 안정성에 맞게 입력한다. 고정된 비상자금 개월 수나 부채비율을 보편적 정답으로 사용하지 않는다.

## 판정

- `financial_clear`: 투자 제외 자금과 손실예산이 완비되고 유동성 공백이 없음.
- `financial_conditional`: 보완 가능한 비상자금·부채·보험·입력 누락이 있어 제한 비중만 검토.
- `stabilization_first`: 투자 확대보다 비상자금·고비용 부채·현금흐름 개선을 우선.
- `financial_blocked`: 확정지출 침해, 연체·강제매도 위험 또는 핵심 입력 부재.

## 100점 조건

- 소득·지출·부채·유동자산·확정지출·목표·손실예산과 기준일이 모두 있음.
- 비상자금·투자자금·단기목표·장기목표가 계정 또는 논리상 분리됨.
- 소득 20% 감소와 예상외 필수지출 시 강제매도 여부를 검증함.
- unknown 0, 중대 차단 0, 모든 계산이 입력 evidence_id와 연결됨.

## 출력

- 재무 체력 판정과 핵심 비율.
- 투자 제외 자금·투자 가능 자금 범위·최대 손실예산.
- 먼저 해결할 부채·유동성·보험·세금 항목.
- 보완 질문과 다음 재평가일.
