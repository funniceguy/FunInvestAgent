# 스킬: 주식 상품·호가 심층 분석

## 목적

최종 전략이 추상적인 섹터나 종목군으로 끝나지 않도록 실제 투자 상품명, 티커, 거래소, 호가, 스프레드, 거래대금, 체결 가능성을 수집하고 주문 실행 가능성을 평가한다.

## 입력

- 시장 범위: 한국, 미국, 글로벌, 특정 섹터.
- 전략 모드: 단타, 가치투자, 혼합, 레버리지.
- 후보 유형: 개별주, ETF, ETN, 인버스, 레버리지, 현금성 ETF.
- 사용자 제약: 계좌 통화, 거래 가능 시장, 세금/수수료, 주문 가능 시간, 최대 손실 한도.
- 데이터 기준 시각과 출처.

## 필수 데이터

| 항목 | 설명 | 우선 출처 |
|---|---|---|
| 상품 식별 | 상품명, 티커, ISIN, 거래소, 통화, 상품 유형 | 거래소, 운용사, DART/SEC |
| 가격 | 현재가 또는 최근 종가, 시가, 고가, 저가, 전일 대비 | 거래소, 공식/신뢰 시장 데이터 |
| 호가 | 최우선 매수호가, 최우선 매도호가, 잔량, 호가 단위 | 거래소, 증권사 HTS/MTS, 공식 시세 벤더 |
| 유동성 | 거래량, 거래대금, 20일 평균 거래대금, 회전율 | 거래소, 공식 시장 데이터 |
| 체결 비용 | 스프레드 금액, 스프레드 %, 예상 슬리피지, 수수료/세금 | 거래소, 증권사, 세무/수수료 표 |
| ETF/ETN | NAV/IOPV, 괴리율, 추적오차, 총보수, LP/시장조성 상태 | 거래소, 운용사, 지표가치 공시 |
| 레버리지 | 배율, 일일 리셋, 복리 효과, 기초지수, 보유 시간 제한 | 운용사, 투자설명서, 감독기관 |
| 공시/규제 | 거래정지, 시장경보, 권리락, 배당락, 주요사항보고서 | KRX, DART, SEC |

## 절차

1. 전략 후보를 실제 매매 가능한 상품으로 매핑한다.
   - 예: `한국 시장 코어` -> KOSPI200 ETF 후보, KOSPI ETF 후보, 현금성 ETF 후보.
   - 예: `미국 대형주 코어` -> S&P500 ETF 후보.
   - 예: `반도체 반등 단타` -> 반도체 대형주 또는 반도체 ETF 후보.
2. 각 후보에 대해 상품명, 티커, 거래소, 통화, 상품 유형을 기록한다.
3. 주문에 쓸 수 있는 가격 데이터는 당일 또는 현재 세션 기준으로만 `actionable` 처리한다.
4. 호가가 실시간으로 확인되지 않으면 `quote_stale` 또는 `quote_unavailable`로 표시하고 신규 주문 결론을 `조건부 검토` 이하로 제한한다.
5. 최우선 매수호가와 매도호가로 스프레드를 계산한다.
   - 스프레드 금액 = 매도호가 - 매수호가.
   - 스프레드 % = 스프레드 금액 / 중간가 x 100.
6. 거래대금과 호가 잔량으로 예상 체결 가능성을 `high`, `medium`, `low`, `blocked`로 분류한다.
7. ETF/ETN은 NAV/IOPV, 괴리율, 추적오차, 총보수, LP 상태를 확인한다.
8. 레버리지/인버스 상품은 일반 후보가 아니라 레버리지 게이트로 넘긴다.
9. 상품별 진입 조건, 손절 기준, 청산 기준, 최대 비중, 금지 조건을 붙인다.
10. 출처, 기준 시각, 지연 여부, 누락 데이터를 기록한다.

## 유동성 판정

| 판정 | 조건 |
|---|---|
| high | 거래대금 충분, 스프레드 좁음, 호가 잔량 충분, 시장조성 정상 |
| medium | 거래 가능하나 분할 주문 필요, 스프레드 또는 잔량 한계 존재 |
| low | 체결 비용이 크거나 거래대금 부족, 소액 관찰만 허용 |
| blocked | 실시간 호가 없음, 거래정지/시장경보, 괴리율 과다, 공시 리스크 |

## 출력

- `stock_product_quote_packet`
- 상품별 실거래 후보 표.
- 호가/스프레드/거래대금 표.
- ETF/ETN NAV·괴리율 표.
- 상품별 실행 가능성 판정.
- 실시간 호가 미확인 시 주문 보류 사유.
- 최종 전략에 넣을 상품명, 티커, 조건부 주문 메모.

## 중단 조건

- 실제 상품명과 티커 없이 신규 진입 전략을 제시해야 하는 경우.
- 호가, 스프레드, 거래대금 중 2개 이상이 없는데 주문 실행을 제시해야 하는 경우.
- ETF/ETN의 NAV/IOPV 또는 괴리율이 없는데 해당 상품을 매수 후보로 제시해야 하는 경우.
- 레버리지 상품인데 일일 리셋, 괴리율, 스프레드, 손실 한도, 보유 시간이 확인되지 않은 경우.
- 거래정지, 시장경보, 미공개정보, 허위정보 의심이 있는 경우.
