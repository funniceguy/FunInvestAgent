# 스킬: 금주 예측 복기·평점 보정

## 목적

금일, 내일, 차주 전략을 만들기 전에 금주 중 작성한 예측·전략·리뷰 자료를 다시 읽고, 성공 예측과 실패 예측을 함께 분석해 다음 전략의 점수 기준과 차단 규칙을 보정한다. 이 스킬은 사후 변명이 아니라 예측 품질을 누적 개선하는 학습 루프다.

## 적용 조건

- 사용자가 금일 전략, 내일 전략, 차주 전략, 다음 주 전략, 예측 적중률 개선, 실패 감소, 보완점 강화를 요청한 경우.
- `funoutput/reports` 또는 `funoutput/reviews`에 이번 주 작성된 주식 전략, 예측 보정, 전망 근거, 성과 검토 자료가 있는 경우.
- 이전 예측의 `prediction_id`, 상승/혼조/하락 확률, 시나리오, 무효화 조건, Verify 점수 기준을 다음 전략에 반영해야 하는 경우.
- 공격적 단기 매매나 레버리지 전략처럼 작은 예측 오류가 큰 손실로 이어질 수 있는 경우.

## 핵심 원칙

- 현재 주 KST 기준 월요일 00:00부터 전략 작성 시각까지의 자료를 기본 복기 범위로 삼는다. 월요일 장 시작 전에는 직전 5거래일 자료를 보조 표본으로 쓴다.
- `stock_cross_market_outcome_audit_skill.md`의 전체 원장을 먼저 읽고, 이 스킬은 그 원장의 금주 이동 창만 갱신한다.
- 이전 보고서가 있으면 반드시 `funoutput/reports`와 `funoutput/reviews`를 검색해 예측 테이블, 근거 카드, 시나리오, Verify 점수를 추출한다.
- prediction_id가 없는 예측 문장은 `legacy_unscorable`, 검증 시각이 지난 미채점 예측은 `matured_unscored`로 유지한다. 둘을 성공률 계산에서 조용히 제외하지 않는다.
- 이전 보고서가 없거나 예측 표본이 부족하면 `복기 표본 없음` 또는 `표본 부족`으로 기록하고, 없는 성공률을 만들지 않는다.
- 성공 예측도 분석한다. 다만 단기 성공 1회만으로 기준 점수를 낮추거나 차단 규칙을 완화하지 않는다.
- 실패 예측은 원인별로 현재 전략의 최소 점수, 가중치, 필수 데이터, 차단 조건 중 하나를 강화해야 한다.
- 사후에 알게 된 정보와 당시 확인 가능했던 정보를 분리한다. 당시 확인 불가능했던 사건은 `새 정보`, 당시 확인 가능했는데 놓친 신호는 `오류`로 분류한다.
- 학습 루프를 통과하지 않은 금일/내일/차주 전략은 `진행 가능`으로 낼 수 없다. 단, 실제로 복기할 이전 보고서가 없으면 그 사실을 명시하고 신규 기준으로 진행한다.

## 필수 입력

- 금주 전략 보고서: `funoutput/reports/YYYY-MM-DD_*stock*strategy*.md`, `*_predictive*`, `*_outlook*`, `*_tactical*` 등.
- 금주 리뷰 보고서: `funoutput/reviews`와 성과 검토 보고서.
- 이전 예측: prediction_id, 대상, 확률, 예상 범위, 핵심 근거, 충돌 근거, 검증 시각, 무효화 조건.
- 실제 결과: 기준 시각 가격, 종가 위치, VWAP, 거래대금, 지수·섹터 대비 상대강도, 외국인·기관·프로그램 수급, 환율, 금리, 해외 선행시장.
- 기존 Verify 기준: 카테고리별 최소 점수와 이전 보정 내역.
- 전수 성과 감사 패킷: 시장·기간·국면별 표본, legacy_unscorable, matured_unscored, 반복 실패 모드.

## 복기 절차

1. 복기 기간을 정한다.
2. 금주 보고서와 리뷰에서 예측 ID, 후보 표시명, 시나리오, 근거 카드, Verify 기준을 추출한다.
3. 전수 원장과 대조해 누락·중복·기한 경과 미채점 예측을 먼저 표시한다.
4. 각 예측의 검증 시각과 실제 결과를 맞춘다.
5. 예측을 `성공`, `부분 성공`, `실패`, `판정 불가`로 분류하고 예측·실행·리스크 결과를 분리한다.
6. 성공 예측은 어떤 근거가 실제로 작동했는지 기록하고 다음 전략에서 유지할 규칙으로만 남긴다.
7. 실패 예측은 데이터 최신성, 기대치 해석, 숨은 기대치, 수급, 환율, 금리, 세계정세, 이벤트, 시간대, 심리, 실행 중 어떤 오류인지 분류한다.
8. 반복 실패와 중대 실패를 찾아 점수 기준 강화안을 만든다.
9. 보정안을 현재 전략의 예측 보정, 거시 민감도, 물량·사이클 게이트, 공격적 단기 매매, Verify 계약에 반영한다.
10. 보정 후에도 핵심 데이터가 없으면 결론을 `조건부 검토`, `보류`, `중단 권고` 중 하나로 낮춘다.

## 평점 강화 규칙

| 조건 | 조정 |
|---|---|
| 같은 실패 원인이 금주 2회 이상 반복 | 관련 카테고리 최소 점수 +3, 해당 데이터 필수화 |
| 같은 실패 원인이 금주 3회 이상 반복 | 관련 카테고리 최소 점수 +5, 중대 차단 조건 추가 |
| 단일 실패라도 손절·무효화 미실행 또는 레버리지 손실로 연결 | 리스크 통제 또는 레버리지 적합성 최소 점수 +5 |
| 외국인·기관·프로그램·환율 오판 반복 | 예측 보정·적중 추적, 공격적 단기 매매 적합성 최소 점수 +3 이상 |
| 숨은 기대치·호재 매도 오판 반복 | 투자자 심리 해석과 거시 원인 민감도 매트릭스 최소 점수 +3 이상 |
| 성공 예측이 2회 이상 같은 근거로 반복 | 해당 규칙 유지, 단 기준 점수 하향 금지 |
| 표본 3개 미만 | 기준 점수 하향 금지, 강화는 severe/critical 실패에만 제한 적용 |

강화된 최소 점수는 다음 1-3개 전략에 적용하고, 이후 리뷰에서 실제 적중률이 개선되었는지 재평가한다. 기본 계약의 최소 점수보다 낮출 수 없다.

## 실패 원인과 강화 대상

| 실패 원인 | 강화할 카테고리 |
|---|---|
| 가격·수급·환율 최신성 부족 | 데이터 신뢰도, 예측 보정·적중 추적 |
| 기대치와 가격 반응 오판 | 투자자 심리 해석, 공격적 단기 매매 적합성 |
| 외국인·기관·프로그램 주도권 오판 | 예측 보정·적중 추적, 공격적 단기 매매 적합성 |
| 금리·환율·세계정세 전달 경로 오판 | 거시 원인 민감도 매트릭스 적합성 |
| 이벤트 일정·실적 발표 후 반응 오판 | 전망 근거 검색·발췌·분석 적합성, 투자자 심리 해석 |
| 시간대·장중 주문 오판 | 미국 장중 시간대 주문 적합성 또는 장중 루틴 |
| 손절·무효화 실행 실패 | 리스크 통제, 실행 가능성 |
| 예측은 맞았지만 진입가가 나쁨 | 상품·호가 실행 상세, 전략 완성도 |

## 출력

- `forecast_learning_loop_packet`
- 복기 기간과 검색한 보고서/리뷰 목록.
- 이전 예측 수, 성공 수, 실패 수, 판정 불가 수.
- 성공 예측에서 유지할 규칙.
- 실패 예측의 원인 분류와 최초 확인 가능 신호.
- 현재 전략에 적용할 점수 기준 강화안.
- 현재 전략에서 차단 또는 보수화할 항목.
- 최종 학습 판정: `calibrated`, `conditional`, `insufficient_sample`, `blocked`, `not_applicable`.

## 중대 차단 조건

- 금일/내일/차주 전략인데 금주 이전 보고서가 존재함에도 검색·복기를 하지 않는다.
- 이전 예측의 성공과 실패를 분리하지 않고 전체 감상으로만 요약한다.
- 실패 원인을 현재 전략의 점수 기준, 차단 조건, 필수 데이터 중 하나로 변환하지 않는다.
- 반복 실패가 있는데 같은 Verify 최소 점수와 같은 전략 논리로 재진입을 제시한다.
- 표본이 부족하거나 단기 성공 1회뿐인데 기준 점수를 낮춘다.
- 사후에만 알 수 있었던 정보를 당시 놓친 신호처럼 처리한다.
- 전수 성과 원장과 금주 복기의 수치가 다르지만 중복·누락 원인을 설명하지 않는다.
