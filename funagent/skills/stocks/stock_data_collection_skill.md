# 스킬: 주식 데이터 수집

## 목적

현재 시점의 주식 전략에 필요한 가격, 공시, 재무, 수급, 거시 데이터를 수집하고 최신성을 표시한다.

## 데이터 카테고리

| 카테고리 | 데이터 | 우선 출처 |
|---|---|---|
| 가격/거래 | 현재가, 거래대금, 변동성, 신고가/신저가 | 거래소, 공식 시장 데이터 |
| 상품/호가 | 상품명, 티커, 거래소, 통화, 최우선 매수·매도호가, 잔량, 스프레드 | 거래소, 증권사 HTS/MTS, 공식 시세 벤더 |
| 공시 | 사업보고서, 분기보고서, 주요사항보고서 | DART, OpenDART, SEC EDGAR |
| 재무 | 매출, 이익, 현금흐름, 부채, ROE, 마진 | 공시 XBRL, 감사 재무제표 |
| 수급 | 기관, 외국인, 내부자, 공매도 | 거래소, 공시 |
| 거시 | 금리, 환율, 물가, 경기 지표 | 중앙은행, 공식 통계 |
| 이벤트 | 실적 발표, 배당, 증자, CB/BW, 규제 | 공시, 감독기관, 회사 발표 |

## 절차

1. 필요한 출처 ID를 `funmemory/sources/source_registry.yaml`에서 고른다.
2. `data_as_of`와 `source_checked_at`을 기록한다.
3. 가격과 거래 데이터는 당일 기준이 아니면 전략 실행 결론에 쓰지 않는다.
4. 공시는 최신 정기보고서와 최근 주요사항보고서를 함께 본다.
5. 뉴스성 정보는 공시 또는 공식 발표로 교차 확인한다.
6. 최종 전략 후보는 `stock_product_quote_depth_skill.md`로 실제 상품명, 티커, 호가, 스프레드, 거래대금, ETF/ETN 괴리율을 별도 패킷으로 만든다.
7. 호가가 실시간이 아니면 `quote_stale`, 호가를 확인하지 못하면 `quote_unavailable`로 표시하고 주문 실행 결론을 제한한다.

## 출력

- 수집 데이터 표.
- 상품·호가 패킷 참조.
- 누락 데이터.
- 오래된 데이터 플래그.
- 전략에 사용 가능한 데이터와 참고용 데이터 구분.
