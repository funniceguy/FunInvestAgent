# 스킬: 주문 실행 및 리스크 통제

## 목적

전략이 실제 주문으로 옮겨질 때 생기는 손실, 세금, 수수료, 유동성, 심리 리스크를 통제한다.

## 주문 전 체크

- 현재가와 호가 스프레드 확인.
- 상품명, 티커, 거래소, 통화, 상품 유형 확인.
- 최우선 매수호가, 최우선 매도호가, 호가 잔량, 호가 기준 시각 확인.
- 거래량과 예상 체결 가능성 확인.
- 거래대금, 20일 평균 거래대금, 스프레드 %, 예상 슬리피지 확인.
- ETF/ETN은 NAV/IOPV, 괴리율, 총보수, 시장조성 상태 확인.
- 손절 주문 또는 수동 대응 가능성 확인.
- 세금, 수수료, 환전 비용 확인.
- 공시, 실적 발표, 배당락, 권리락, 거래정지 가능성 확인.
- 투자자 심리 게이트가 신규 진입을 허용하는지 확인.
- 공격적 단기 매매 확률 게이트가 `aggressive_allowed` 또는 `tactical_allowed`인지 확인.
- 기관/외국인/프로그램 수급과 환율이 주문 방향과 충돌하지 않는지 확인.
- 국내·미국 전수 감사에 matured_unscored 과다 또는 동일 실패 반복이 없는지 확인.
- 복수 호가 스냅샷, 실제 체결, VWAP/AVWAP, 매물대, 잠재 공급을 확인하고 물량 게이트가 `flow_clear` 또는 `flow_caution`인지 확인.
- 급등·순환 사이클 클래스, 신뢰도, 현재 단계와 주문 보유 시간이 일치하는지 확인.
- 자본보전·손실회복 게이트가 `capital_preservation_clear`인지 확인. conditional은 조건 보완 전 주문 보류.
- 계획 손절, 갭·슬리피지 스트레스, 유효 위험폭, 위험 기준 포지션 상한 확인.

## 실행 규칙

- 손실 한도를 넘긴 포지션은 물타기보다 논리 재검토를 먼저 한다.
- 단타 계획이 장기투자 핑계로 바뀌면 실패 거래로 기록한다.
- 가치투자 계획이 단기 손익 때문에 흔들리면 thesis review로 분리한다.
- 시장 급변 시 신규 진입보다 기존 위험 축소를 우선 검토한다.
- 심리 게이트가 `watch_only` 또는 `avoid`이면 주문 실행을 보류한다.
- 공격적 단기 매매 확률 게이트가 `watch_only` 또는 `blocked`이면 신규 진입을 보류한다.
- 수급 또는 환율이 진입 방향과 충돌하면 포지션 크기를 줄이거나 관찰로 낮춘다.
- 호가, 스프레드, 거래대금 중 2개 이상이 없으면 신규 주문을 보류한다.
- 호가 스프레드 또는 괴리율이 과도하면 시장가 주문을 금지하고 지정가·분할 주문 또는 관찰로 낮춘다.
- 물량 게이트가 watch_only, blocked, unknown이면 신규·추가 주문과 시장가 주문을 금지한다.
- 급등 exhaustion, 순환 mature_rotation, distribution_markdown이면 신규 진입보다 축소·청산·재평가를 우선한다.
- 손절가는 체결 보장 가격으로 사용하지 않는다. 갭·슬리피지 스트레스 손실이 한도를 넘으면 포지션을 축소하거나 주문을 보류한다.
- 자본보전 게이트가 conditional, watch_only 또는 blocked이면 신규·추가 주문과 레버리지를 금지한다.
- 구조적 훼손 또는 손실 복구 목적 거래는 손절·물타기 방식과 무관하게 주문을 금지한다.

## 출력

- 주문 가능 여부.
- 주문 방식 메모.
- 상품별 호가·스프레드·거래대금 메모.
- 체결 가능성 판정: high / medium / low / blocked.
- 손실 한도.
- 스트레스 손실, 유효 위험폭, 포지션 상한, 필요 회복수익률·기간.
- 중단 조건.
- 사후 기록 항목.
- 체결가, 슬리피지, 비용, MFE/MAE, 손절 준수와 prediction_id 연결.
