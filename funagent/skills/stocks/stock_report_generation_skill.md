# 스킬: 주식 전략 리포트 생성

## 목적

주식 분석 결과를 현재 행동과 자금 배분이 보이는 리포트로 만든다.

## 템플릿

- 신규 전략: `funoutput/templates/stock_strategy_report.md`
- 성과 검토: `funoutput/templates/stock_strategy_review.md`

## 작성 순서

1. 입력 데이터와 가정을 먼저 쓴다.
2. 차주/미래 전망 요청이면 검색 커버리지, 근거 카드, 기본/강세/약세 시나리오, 전략 방향을 쓴다.
3. 시장 국면과 데이터 기준일을 쓴다.
4. 실제 상품명, 티커, 거래소, 통화, 호가, 스프레드, 거래대금, ETF/ETN 괴리율을 쓴다.
5. 투자자 심리 해석 게이트를 쓴다.
6. 저평가/전망/성장형/미래형 후보 요청이면 확장형 유니버스 섹터 맵을 쓰고, 사용자 언급 종목 외 후보와 제외 사유를 포함한다.
7. 공격적 단기 매매 요청이면 확률 게이트의 기대치, 가격 반응, 거래대금, 기관/외국인/프로그램 수급, 환율, 상대강도, 손익비 점수를 쓴다.
8. 레버리지 요청이 있으면 레버리지/3배 단타 게이트를 쓴다.
9. 단타 전략과 가치투자 전략을 분리한다.
10. 슬리브별 자금 배분을 쓴다.
11. 현재 행동, 보류 조건, 중단 조건을 쓴다.
12. 성과 검토 일정을 쓴다.
13. `stock_verification_contract_skill.md`로 최소 2회 Verify 평점 검증을 수행한다.
14. 레버리지 요청이 있으면 `stock_leverage_verification_contract_skill.md`로 추가 검증한다.
15. 기준 미달 항목을 수정하고 반복 검증 기록을 남긴다.
16. 출처와 검증 체크리스트를 채운다.

## 출력 품질

- 사용자가 바로 확인할 액션이 있어야 한다.
- 실제 매매 후보는 상품명, 티커, 거래소, 통화, 상품 유형을 함께 써야 한다.
- 주문 실행 후보는 현재가 또는 최근 종가, 최우선 매수·매도호가, 스프레드 %, 거래대금, 데이터 기준 시각을 포함해야 한다.
- 실시간 호가가 없으면 `호가 미확인으로 신규 주문 보류`를 명시한다.
- ETF/ETN 후보는 NAV/IOPV, 괴리율, 총보수, 유동성 판정을 포함해야 한다.
- 차주/미래 전망 리포트는 공식/1차 출처 검색, 근거 카드, 세계정세·경제·수급·심리 시나리오, 무효화 조건을 포함해야 한다.
- 전망 합성 판정이 `watch_only` 또는 `blocked`이면 미래 전망을 신규 진입의 핵심 근거로 쓰지 않는다.
- 공격적 단기 매매 리포트는 기대치 갭, VWAP/가격 반응, 거래대금, 기관/외국인/프로그램 수급, 환율, 손익비, 1회 손실 한도를 포함해야 한다.
- 공격적 단기 매매 확률 게이트가 `watch_only` 또는 `blocked`이면 신규 진입 전략을 쓰지 않는다.
- 확장형 유니버스 리포트는 섹터별 대표 후보, 대체 후보, 제외 후보, `좋은 산업/좋은 진입가` 분리를 포함해야 한다.
- 사용자 언급 종목만 평가한 경우에는 `전체 시장 또는 확장 유니버스 스크리닝 아님`이라고 명시하고, Verify에서 남은 약점으로 기록한다.
- 실제 금액이 없으면 비중으로 표현한다.
- 매수/매도 단정 대신 조건부 실행 계획으로 쓴다.
- 실패했을 때 무엇을 바꿀지 미리 정의한다.
- 전체 Verify 가중 점수 88점 미만이거나 중대 차단 조건이 있으면 `진행 가능` 결론을 쓰지 않는다.
- 최종 리포트에는 카테고리별 점수, 수정 내역, 남은 약점, 최종 판정을 포함한다.
- 심리 게이트가 `watch_only` 또는 `avoid`이면 신규 진입 전략을 쓰지 않는다.
- 3배 레버리지 게이트가 `watch_only` 또는 `blocked`이면 신규 진입 전략을 쓰지 않는다.
