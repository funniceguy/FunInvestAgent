# 스킬: 코인 포트폴리오·적립·실행

## 목적

자본 생존 게이트를 통과한 위험자본 안에서 코어·위성·수익·전술 슬리브와 24시간 실행 규칙을 설계한다.

## 슬리브

- `liquidity_reserve`: 코인 외부의 예금·MMF·단기채. 비상금과 분리.
- `crypto_core_spot`: 검증된 유동성·보안·분산도가 높은 현물 적립 후보.
- `crypto_satellite`: 실사 통과 알트·프로토콜의 제한 비중.
- `crypto_yield`: 별도 수익 게이트를 통과한 스테이킹·DeFi.
- `crypto_tactical`: 손실예산과 만료 시간이 있는 단기 현물.

## 실행 규칙

- 사용자 정보가 부족하면 실제 비중 대신 손실예산 산식과 질문을 제시한다.
- 적립은 달력 기반 기본 납입과 밴드 기반 보수화를 분리한다. 가격 급등만으로 납입액을 확대하지 않는다.
- 신규 자금으로 목표 미달 슬리브를 먼저 채우고, 매도 리밸런싱은 세금·수수료·유동성을 확인한다.
- 거래당 수량은 스트레스 손실이 거래·슬리브·전체 한도 중 최솟값을 넘지 않게 산정한다.
- 시장가 주문은 깊이와 슬리피지가 검증된 소액 외 금지한다. 분할 지정가와 최대 허용 스프레드를 기록한다.
- 수면 시간에는 계획된 적립 외 신규 재량 진입을 금지하고, 알림·취소·비상 출금 절차를 둔다.
- 장기 보유는 거래소 집중과 자기수탁 운영 실패를 함께 비교해 보관 방식을 선택한다.

## 매도·중단

- 목표 비중 상단 초과, 논리 훼손, 보관·규제·상폐 위험, 잠금해제·희석 변화, 목표일 도래 시 축소한다.
- 가격 하락만으로 자동 물타기하지 않는다. 실사·수급·보관 게이트를 다시 통과한 경우에만 신규 적립으로 조정한다.
- 디페그, 출금 중단, 컨트랙트 변경, 브리지 사고, 오라클 이상은 수익률보다 접근권 보전을 우선한다.
