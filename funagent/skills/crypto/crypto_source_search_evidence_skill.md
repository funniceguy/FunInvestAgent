# 스킬: 코인 공식 정보 검색·근거 카드

## 목적

현재 전략에 필요한 시장·온체인·파생·프로토콜·보관·규제 정보를 검색하고 사실, 계산, 해석을 분리한다.

## 출처 우선순위

1. 법령·감독기관·규제 거래소·공식 공시·법원 문서.
2. 블록체인 노드·공식 탐색기·프로토콜 저장소·거버넌스·토큰 컨트랙트.
3. 실제 실행 거래소의 공개 API와 상품 명세, 규제 파생시장·CFTC 자료.
4. 독립 분석 제공자의 방법론 공개 데이터.
5. 뉴스·소셜은 사건 탐색과 심리 보조에만 사용.

## 필수 검색 묶음

- 가격·거래대금·호가·스프레드: 실행 거래소와 독립 현물 시장 1곳 이상.
- 파생: 선물 베이시스, 펀딩, OI, 옵션 IV/skew, 청산과 상품 명세.
- 온체인: 공급·발행·소각, 거래소 흐름, 집중도, 네트워크 사용과 기준 블록.
- 프로토콜: 컨트랙트, 감사 범위, 버그바운티, 관리자키·업그레이드, 오라클·브리지, 사고 이력.
- 토큰: 유통량·FDV·다음 30/90일 잠금해제·재단/팀/VC 지갑.
- 보관·법률: 거래소 법인·관할·고객자산 지위·출금 정책·세금·규제 기준일.

## 신선도

- 단기 주문 가격·호가·스프레드: 15분.
- 펀딩·OI·청산·베이시스: 1시간.
- 일간 전략의 현물·온체인·ETF/ETP 흐름: 24시간.
- 공급·잠금해제·거버넌스 일정: 7일 이내 재확인, 이벤트 직전 당일 확인.
- 준비금·감사·법률·세금: 최신 원문과 효력일을 기록하고 30일 내 변경 여부 재검색.

## 근거 카드

각 카드는 `evidence_id`, `source_id`, URL, published_at, data_as_of, checked_at, 자산 표시명·네트워크·컨트랙트, 사실, 계산, 해석, 충돌 근거, 한계를 포함한다. 온체인 수치는 기준 블록과 라벨링 방법론을 기록한다.

## 차단

- 소셜 게시물, 인플루언서, 거래소 홍보, 백서만으로 수익·안전성·채택을 확정.
- 출처 시각과 데이터 시각이 없거나 가격 통화·거래소가 불명확.
- 단일 데이터 제공자의 주소 라벨·청산 추정치를 확정 사실로 사용.
