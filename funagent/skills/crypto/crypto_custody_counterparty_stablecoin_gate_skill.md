# 스킬: 코인 보관·거래상대방·스테이블코인 게이트

## 목적

가격 방향과 무관하게 자산 접근권을 잃는 거래소, 수탁, 지갑, 브리지, 스테이블코인 위험을 차단한다.

## 거래소·수탁 검증

- 계약 법인, 관할, 등록·신고 상태, 고객자산 법적 지위와 도산 시 청구 순위.
- 원화·법정화폐 예치금과 가상자산 보관의 보호 범위를 분리.
- 고객자산 분리, 준비금·부채·감사 범위, 보험의 피보험자·면책·한도.
- 입출금 상태, 네트워크별 출금, 소액 테스트, 주소 허용목록, 2FA·패스키, API 권한.
- 준비금 증명은 특정 시점 자산 스냅샷이며 총부채·우선순위·기간 중 운용을 증명하지 못한다고 기록.

## 지갑·브리지 검증

- 키 소유자, 백업·복구·상속, 펌웨어·서명 화면, 피싱·승인 권한, 테스트 전송.
- 브리지·래핑 자산은 원자산 수탁, 검증자·관리자키, 상환 경로와 사고 이력을 확인.

## 스테이블코인 검증

- 발행자·관할·법적 상환권·대상 고객·최소 상환액·수수료·시간.
- 준비자산 구성, 수탁기관, 만기·유동성·담보 초과 여부, 검증 제공자·기준일·검증 수준.
- 1일/7일 가격 괴리, 거래소별 괴리, 유동성 깊이, 대규모 상환 스트레스.
- 알고리즘·암호담보·법정화폐 담보를 분리하고 연쇄청산·오라클·거버넌스 위험을 추가한다.

## 판정

- `clear`: 접근권·법적 구조·유동성·복구가 확인되고 집중 한도 내.
- `conditional`: 일부 한계가 있으나 소액·분산·짧은 체류와 출금 테스트로 제한.
- `watch_only`: 핵심 법적·부채·상환·보관 정보 unknown.
- `blocked`: 출금 중단, 중대한 디페그, 지급불능·해킹·제재 위험, 컨트랙트 불일치.
